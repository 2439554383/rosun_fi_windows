# 入金功能固定配置说明

## 🔧 写死配置

为了简化用户操作，以下信息已经写死在代码中：

### 1. 银行信息
```dart
final fixedBankId = 1; // 固定银行ID
final fixedBankName = '【中国香港】花旗银行(4546)'; // 固定银行名称
```

### 2. 凭证信息
```dart
final fixedTransferProof = ['transfer_proof_${DateTime.now().millisecondsSinceEpoch}.jpg'];
```
**注意**: 后端期望 `transferProof` 是数组格式，不是字符串

## 📱 UI变化

### 银行选择区域
- **原来**: 可点击的下拉选择框
- **现在**: 固定的银行信息显示框，带有银行图标

### 凭证上传区域
- **原来**: 可点击的上传区域
- **现在**: 显示"凭证已自动生成"的固定区域

## 🎯 用户体验

### 优势
1. **简化操作** - 用户只需输入金额即可
2. **减少错误** - 避免用户选择错误的银行
3. **提高效率** - 减少用户操作步骤
4. **统一标准** - 所有入金都使用同一银行

### 流程
1. 用户输入入金金额
2. 系统自动使用固定银行信息
3. 系统自动生成凭证文件名
4. 用户点击提交即可

## 🔄 如需修改

### 修改银行信息
在 `new_deposit_ctrl.dart` 的 `submitDeposit()` 方法中：
```dart
// 修改银行ID
final fixedBankId = 2; // 改为其他银行ID

// 修改银行名称
final fixedBankName = '【中国香港】汇丰银行(1234)'; // 改为其他银行
```

### 修改凭证格式
```dart
// 修改凭证文件名格式（注意：必须是数组）
final fixedTransferProof = ['deposit_proof_${DateTime.now().millisecondsSinceEpoch}.png'];
```

## 📋 当前配置详情

| 配置项 | 当前值 | 说明 |
|--------|--------|------|
| 银行ID | 1 | 后端银行标识 |
| 银行名称 | 【中国香港】花旗银行(4546) | 显示给用户的银行信息 |
| 凭证格式 | ["transfer_proof_[时间戳].jpg"] | 自动生成的凭证文件名数组 |
| 入金方式 | 1 | 存入国际电汇 |
| 币种 | HKD | 港币 |

## ⚠️ 注意事项

1. 确保后端支持银行ID为1的银行信息
2. 凭证文件名格式需要与后端期望的格式匹配
3. 如需支持多银行，需要重新启用银行选择功能
4. 凭证文件需要在实际业务中处理上传逻辑
